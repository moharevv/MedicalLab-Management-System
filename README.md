# New_AppLabaratory

Desktop-приложение для автоматизации работы лаборатории. Проект реализован на `WPF` и моделирует основные процессы лабораторной информационной системы: авторизацию сотрудников, регистрацию пациентов, оформление заказов, работу с биоматериалом и интеграцию с анализаторами.

## Возможности

- авторизация пользователей по ролям;
- защита входа с капчей после неудачных попыток;
- просмотр истории авторизаций;
- регистрация пациентов;
- создание заказов на исследования;
- выбор нескольких услуг в рамках одного заказа;
- генерация штрих-кодов для пробирок;
- сканирование штрих-кодов с камеры и из PDF;
- отправка исследований на анализаторы;
- отображение прогресса выполнения;
- подтверждение и отклонение результатов исследований.

## Роли пользователей

- `Администратор`
  Просмотр журнала входов и базовая работа с административным окном.

- `Бухгалтер`
  Отдельное окно роли бухгалтера.

- `Лаборант`
  Регистрация пациентов, оформление биоматериала и создание заказов.

- `Лаборант-исследователь`
  Работа с анализаторами, контроль прогресса и обработка результатов.

## Технологии

- `C#`
- `WPF`
- `.NET Framework 4.7.2`
- `SQL Server LocalDB`
- `packages.config`

### Используемые библиотеки

- `AForge`, `AForge.Video`, `AForge.Video.DirectShow` — работа с камерой;
- `ZXing.Net` — распознавание штрих-кодов;
- `Aspose.BarCode`, `Aspose.PDF` — генерация штрих-кодов и PDF;
- `Newtonsoft.Json`, `System.Web.Extensions` — сериализация и обмен данными.

## Интеграция с анализаторами

Приложение работает с локальным API анализаторов:

- `http://localhost:5000/api/analyzer/Biorad`
- `http://localhost:5000/api/analyzer/Ledetect`

Реализован следующий сценарий:

1. Приложение отправляет исследование на анализатор.
2. Анализатор принимает запрос и начинает обработку.
3. Приложение опрашивает API по прогрессу выполнения.
4. После завершения результат сохраняется в базу данных и отображается в интерфейсе.

Поддерживается массовая отправка нескольких услуг, если они относятся к одному заказу.

## База данных

В проекте используется локальная база данных `SQL Server LocalDB`.

Файлы базы:

- `New_AppLabaratory/Database/base_laboratory.mdf`
- `New_AppLabaratory/Database/base_laboratory_log.ldf`

Подключение выполняется через `|DataDirectory|`, поэтому база автоматически подхватывается из папки проекта.

```text
New_AppLabaratory/
├─ New_AppLabaratory/
│  ├─ Administrator/          # окно администратора
│  ├─ Analyzer/               # анализатор и конфигурация API

│  └─ packages.config
```

## Ключевые экраны

- `MainWindow`
  Авторизация и переход в окно соответствующей роли.

- `WelcomeBio`
  Создание заказа, выбор пациента, биоматериала и услуг.

- `AddPacient`
  Регистрация нового пациента с базовой валидацией данных.

- `CreateBarcode` и `ScannerBarcode`
  Генерация и считывание штрих-кодов.

- `AnalyserWork`
  Работа с анализаторами, прогресс выполнения, подтверждение и отклонение результатов.

## Запуск проекта

### Требования

- Windows
- Visual Studio с поддержкой `.NET Framework 4.7.2`
- установленный `SQL Server LocalDB`

### Как запустить

1. Открыть решение `New_AppLabaratory.slnx`.
2. Убедиться, что доступны NuGet-пакеты из папки `packages` или выполнить восстановление пакетов.
3. Проверить наличие файлов базы данных в папке `New_AppLabaratory/Database`.
4. При необходимости запустить `New_AppLabaratory/Analyzer/LIMSAnalyzers.exe`.
5. Запустить проект `New_AppLabaratory`.
